# Programming Language Specifications for ML/AI Training

This document provides comprehensive specifications for each programming language in our training dataset, focusing on syntax patterns, paradigms, and features that AI coding agents should understand.

## Language Categories and Paradigms

### Object-Oriented Languages
- **Java**: Pure OOP, static typing, compiled to bytecode
- **C#**: .NET ecosystem, similar to Java with additional features
- **C++**: Multi-paradigm with strong OOP support, manual memory management
- **Python**: Multi-paradigm with strong OOP support, dynamic typing
- **JavaScript**: Prototype-based OOP, dynamic typing
- **TypeScript**: JavaScript with static typing, compiled to JavaScript
- **Kotlin**: Modern JVM language, interoperable with Java
- **Swift**: Modern language for Apple platforms
- **Dart**: Object-oriented, used for Flutter development
- **Solidity**: Contract-oriented for blockchain development

### Functional Languages
- **Haskell**: Pure functional, lazy evaluation, strong type system
- **Scala**: Functional/OOP hybrid on JVM
- **R**: Functional-oriented for statistical computing

### Systems Programming Languages
- **C**: Procedural, low-level, manual memory management
- **Rust**: Systems programming with memory safety
- **Go**: Simple, concurrent programming focus

### Scripting Languages
- **Python**: General-purpose scripting and development
- **JavaScript**: Web scripting and full-stack development
- **Ruby**: Object-oriented scripting
- **Lua**: Lightweight scripting, often embedded
- **Shell/Bash**: System administration and automation
- **PHP**: Web development scripting

### Domain-Specific Languages
- **SQL**: Database query and management
- **R**: Statistical computing and data analysis
- **MATLAB**: Mathematical computing and engineering
